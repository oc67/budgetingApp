About

Find below a description of the product features.

CRUD:

This budgeting app has a basic CRUD functionality for creating and maintaining monthly budgets.


FORMS:

Budgets feature headers and expense lines, and a field for the total income of the period.
Django crispy forms are used when creating and updating budgets.
Budget expenses need a name, category, quantity and price per unit.
The app will display the total price per line once the budget is saved.


JAVASCRIPT:

Success message:
Using Javascript, success messages are shown when budgets are created or transfered successfully.
Also, validation errors are also shown on other views of the app such as on the budget transfer
menu.
Plots:
A basic feature to display a plot with the aggregated costs per category has been created.



AUTHENTICATION:

In order to access the website, visitors need to sign up and log in.


AUTHORISATION:

Users can only view and update their own budgets.


REST FRAMEWORK - API:

Added basic API which reveals budget header data.
Authentication: required to access the API.
Authorisation: only budget owners can view, delete, or edit information revealed at the API endpoints.


EXTERNAL API CONSUMPTION:

An external currency exchange API is consumed to convert budget values to two other currencies, USD and
EUR, and display such values on several menus such as the list and detail budget views.


EXPORTS TO CSV:

Users can export individual budgets to .csv.


TRANSFERS AMONG USERS:

Users can send budgets to other users. Inbound and outbound transactions are recorded on the notifications menu.


ROLE-BASED ACCESS CONTROL:

Original budget owners retain view permissions after transfering their budgets.
